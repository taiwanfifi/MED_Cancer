## 一、為什麼做這個研究？癌症治療的根本困境

### 癌症為什麼難治？

如果你寫過程式就會知道，bug 最難修的不是「程式完全不能跑」，而是「程式大部分時候能跑，但偶爾會出問題」。癌症就是這種 bug。

癌症的本質是**基因突變**。人體有大約 20,000 個基因，它們像程式碼一樣控制細胞的行為 — 何時分裂、何時停止、何時自我銷毀。當關鍵基因發生突變（你可以想像成程式碼被改壞了），細胞就開始不受控制地增殖，這就是腫瘤。

但問題在於：

1. **異質性**：同一個病人的腫瘤裡，不同細胞的突變可能不一樣（就像同一個 codebase 不同 branch 有不同的 bug）
2. **抗藥性**：你用藥殺了 99% 的癌細胞，剩下 1% 如果剛好不受這種藥影響，它們就會重新長回來（就像你修了一個 bug 但引入了另一個）
3. **正常細胞也受傷**：大部分化療藥物是「殺所有快速分裂的細胞」，所以頭髮細胞、腸道細胞也會被殺死 — 這就是化療副作用的來源

### 精準醫療的目標

理想的癌症治療應該是：**只殺癌細胞，不傷正常細胞**。

問題是：癌細胞是從正常細胞變來的，它們大部分的基因和正常細胞一模一樣。要找到「只存在於癌細胞、不存在於正常細胞」的弱點，這就是整個精準醫療領域在做的事。

而我們的研究，就是用**計算的方法**，從海量的基因體資料中，系統性地搜索這些弱點。

## 二、核心概念：什麼是「合成致死」？

### 用程式的比喻來理解

想像你在維護一個線上服務，這個服務有兩個備援系統（Redundancy）— 系統 A 和系統 B。

- **系統 A 壞了**：沒問題，系統 B 接手，服務照常運作
- **系統 B 壞了**：沒問題，系統 A 接手，服務照常運作
- **系統 A 和 B 同時壞了**：服務掛掉 (crash)

這就是**合成致死 (Synthetic Lethality, SL)** 的概念：

> 兩個基因，單獨失去任何一個，細胞都能活。但**兩個同時失去**，細胞就會死亡。

### 為什麼這對治癌有用？

癌細胞因為突變，已經**天生就少了一個備援系統**（比如 BRCA1 基因壞了）。如果我們找到它的 SL 夥伴基因（比如 PARP1），然後用藥物去抑制 PARP1，那麼：

- **癌細胞**：BRCA1 已經壞了 + PARP1 被藥物抑制 = 兩個都沒了 → 死亡
- **正常細胞**：BRCA1 正常 + PARP1 被藥物抑制 = 還有一個在 → 存活

這就實現了「只殺癌細胞、不傷正常細胞」的目標。

### 臨床上已經成功的案例

這不是純理論。2014 年 FDA 批准了 **Olaparib（PARP 抑制劑）** 用於 BRCA 突變的卵巢癌 — 這正是合成致死的臨床應用。從 2005 年 Bryant 等人在 *Nature* 上發表 BRCA-PARP 合成致死的論文，到 FDA 批准，花了 9 年。

**我們的研究目標**：不是只研究一對 SL 關係，而是**系統性地掃描整個基因體**，找出所有可能的 SL 弱點，橫跨 33 種癌症類型。

## 三、我們的策略：用資料科學系統性地掃描癌症弱點

### 整體思路

如果用 AI / 資料科學的術語來描述，我們做的事情本質上是一個**大規模的 Feature Engineering + Pattern Mining + Multi-Source Data Fusion** 問題：

```
輸入：
  - 11,428 位癌症病人的基因表達 + 突變 + 臨床資料 (TCGA)
  - 1,086 個癌細胞株的基因敲除實驗結果 (DepMap)
  - 5,826 個藥物-基因交互作用 (DGIdb)
  - 獨立驗證資料集 (GEO)

處理：
  - 24 個計算實驗，從不同角度分析
  - 合成致死發現、存活分析、差異表達、藥物匹配、
    免疫微環境、表觀遺傳、圖神經網路、LASSO 回歸...

輸出：
  - 126 個新發現的 SL 靶點
  - 4,750 個基因的多證據綜合評分
  - 61% 病人可對應到的精準用藥建議
  - 一個經過獨立驗證的 6 基因預後簽名
  - 15 篇論文的數據基礎
```

### 為什麼是「計算研究」？

做生物實驗（濕實驗）非常昂貴且緩慢 — 驗證一對 SL 關係可能需要數週的細胞培養 + 基因編輯。而我們的計算方法可以在幾小時內掃描**數十萬對基因組合**，從中篩選出最值得用濕實驗驗證的靶點。

這就像：你不會隨機測試所有可能的 hyperparameter 組合，而是先用 grid search 或 Bayesian optimization 縮小範圍，再重點測試。我們的計算研究就是癌症治療的「hyperparameter search」。

## 四、資料從哪來？我們用了什麼「原料」

### 核心資料集

| 資料集 | 白話解釋 | 規模 | 類比 |
|--------|---------|------|------|
| **TCGA** | 美國國家癌症研究所收集的「癌症病人百科全書」— 每位病人的腫瘤被全面定序，記錄哪些基因表達高、哪些有突變、病人活了多久 | 11,428 位病人，33 種癌症 | 像 ImageNet 之於電腦視覺 |
| **DepMap** | 美國 Broad 研究所對 1,086 個癌細胞株做的「基因敲除大全」— 把每個基因逐一用 CRISPR 剪掉，看細胞會不會死 | 1,086 細胞株 × 17,386 基因 | 像 A/B testing 但規模是百萬級 |
| **DGIdb** | 藥物-基因交互作用資料庫 — 記錄哪些藥物能影響哪些基因的功能 | 5,826 個交互作用，3,344 種藥物 | 像是一本「藥物對照表」 |
| **OpenTargets** | 歐洲生物資訊研究所維護的疾病-藥物-靶點知識圖譜 | 癌症相關藥物關聯 | 像是疾病的 Knowledge Graph |
| **GEO** | 獨立研究團隊發表的基因表達資料集，我們用來做外部驗證 | GSE20685 (n=327), GSE72094 (n=398) | 像是 held-out test set |

### 關鍵概念：為什麼 DepMap 是我們的核心？

DepMap 的 CRISPR 篩選資料，本質上是一張巨大的矩陣：

```
            基因1  基因2  基因3  ... 基因17386
細胞株 A    -0.5   -2.1   0.3   ...
細胞株 B    -1.8   -0.1   -0.7  ...
細胞株 C     0.1   -3.2   -1.5  ...
...
細胞株 1086
```

數值代表「敲掉這個基因後，這個細胞株的生長受多大影響」。負值越大 = 越依賴這個基因（敲掉就活不了）。

**合成致死的發現方式**：如果「攜帶 KRAS 突變的細胞株」普遍很依賴某個基因 X（負值很大），但「KRAS 正常的細胞株」不太依賴 X — 那 X 就是 KRAS 的 SL 夥伴。用 AI 的術語說，我們在做的就是 **conditional dependency analysis**。

## 六、五大突破性發現

### 發現 1：FGFR1 — 所有證據都指向它

**白話說**：我們從不同角度分析了 4,750 個基因，FGFR1 在每一個維度都排在前面 — 有 35 種已核准藥物可以打它、細胞實驗證實它很重要 (AUC=0.84)、在腫瘤中表達異常、文獻中被嚴重低估。

**為什麼重要**：這代表 FGFR 抑制劑（像是 Infigratinib）有很強的理由被應用到更多癌症類型。目前 FGFR 抑制劑主要用在膽管癌和膀胱癌，我們的數據支持它的泛癌潛力。

**CS 類比**：就像做 feature importance 分析，發現某個 feature 在 random forest、gradient boosting、logistic regression 中都排前三 — 你可以很有信心地說這個 feature 是真的重要。

### 發現 2：GPX4 — 鐵死亡的泛癌弱點

**白話說**：GPX4 是一個負責保護細胞不被「氧化鐵生鏽」殺死的基因。我們發現它跟 5 個最常見的癌症驅動基因（BRAF、CTNNB1、EGFR、ERBB2、VHL）都存在合成致死關係。更重要的是 — **目前沒有任何已核准藥物能直接抑制 GPX4**。

**鐵死亡是什麼**：細胞有一種特殊的死法叫「鐵死亡 (ferroptosis)」— 當細胞內的鐵離子累積太多、脂質被氧化破壞，細胞就會死亡。GPX4 是對抗鐵死亡的最後一道防線。

**PTEN 突變讓這更致命**：我們發現 PTEN（一個常見的腫瘤抑制基因）突變的癌細胞，極度依賴穀胱甘肽 (GSH) 合成路徑來抵抗鐵死亡。效應量 Cohen's d 達到 −2.00（統計上算是「極大效應」），p 值 < 10⁻¹⁹。

**CS 類比**：就像你發現某個微服務的 single point of failure — 大部分時候有備援所以沒事，但當備援也被拿掉時，系統必然崩潰。GPX4 就是癌細胞的那個 single point of failure。

### 發現 3：SWI/SNF → PRC2 表觀遺傳重塑

**白話說**：SWI/SNF 和 PRC2 是兩個負責「管理基因開關」的蛋白質複合體（你可以想像成兩個不同的 config manager）。正常情況下它們互相制衡。但當 SWI/SNF 壞掉（突變）時，PRC2 就變得沒人管 → 癌細胞開始依賴 PRC2 來存活。

我們找到了 25 對這樣的「重塑關係」。特別是 EZH2（PRC2 的核心成員）有 5 個來自 SWI/SNF 的重塑夥伴 — 而 **Tazemetostat（EZH2 抑制劑）已經被 FDA 核准**用於其他適應症。

**為什麼重要**：這意味著已經有現成的藥可以用，不需要從零開發。

### 發現 4：6 基因預後簽名通過獨立驗證

**白話說**：我們用 LASSO 回歸從 SL 靶點基因中篩選出 6 個基因的組合（BRAF、KIF14、CDK6、ORC6、CHAF1B、MAD2L1），可以預測肺腺癌病人的預後。

關鍵數字：
- 交叉驗證 C-index = 0.733（1.0 是完美預測，0.5 是亂猜）
- **獨立肺腺癌世代驗證：C-index = 0.734, p = 6.8×10⁻⁴**

**CS 類比**：在一個 dataset 上訓練，在完全不同的 dataset 上 inference 還能維持幾乎一樣的 accuracy — 這代表模型沒有 overfit，是真的學到了 pattern。

**為什麼獨特**：這是第一個完全從合成致死網路衍生出來、且經過獨立驗證的預後簽名。

### 發現 5：61% 的癌症病人可以用 SL 策略治療

**白話說**：我們建了一條完整的 pipeline：病人突變 → 找到 SL 夥伴 → 對應已核准藥物。在 1,577 位 TCGA 病人中：
- **965 位 (61%)** 的突變可以透過 SL 策略靶向
- **210 位 (13%)** 有現成的已核准藥物可以直接使用

另外，我們發現 BCL2（一個抗凋亡基因）在「免疫冷」的乳癌腫瘤中顯著上調。這意味著 Venetoclax（BCL2 抑制劑）+ 免疫檢查點抑制劑的組合，可能讓原本對免疫治療無反應的腫瘤重新產生反應。

## 七、技術方法總覽（給 CS / AI 背景的人）

### 我們用的演算法和框架

| 類別 | 方法 | 用在哪 | 備註 |
|------|------|-------|------|
| **統計檢定** | Welch's t-test, Mann-Whitney U, Fisher exact | SL 發現 (Exp 2)、差異表達 (Exp 3) | 基本功，但在生物資訊中是核心工具 |
| **存活分析** | Cox Proportional Hazard, Kaplan-Meier, Log-rank test | Exp 1, 6, 8, 11, 22 | 醫學資料的特殊之處：有「審查資料 (censored data)」— 病人追蹤結束時還活著 |
| **正規化迴歸** | LASSO Cox (L1 regularization) | 預後簽名 (Exp 22)、生物標記 (Exp 9) | 跟你們 ML 裡用的 L1 完全一樣的東西 |
| **圖神經網路** | HeteroSAGE (heterogeneous graph) | 藥物再利用 (Exp 5) | 異質圖：藥物節點 + 基因節點 + 疾病節點，做 link prediction |
| **深度學習存活** | DeepSurv | 存活預測 (Exp 8) | Cox loss function 的神經網路版本 |
| **聚類** | Consensus Clustering (KMeans × 50 bootstraps) | 病人分群 (Exp 16) | 穩健版的 KMeans，80% subsampling 重複 50 次 |
| **免疫反卷積** | ssGSEA-like scoring | 免疫分析 (Exp 6, 19) | 從 bulk RNA-seq 推算免疫細胞組成（類似 NMF 的概念） |
| **隨機森林** | Random Survival Forest (RSF) | 存活預測 (Exp 8) | ensemble learning 在存活分析的應用 |
| **文獻探勘** | PubMed API batch query | 文獻驗證 (Exp 13) | 350 次 API call，計算「新穎性差距 (Novelty Gap)」 |
| **網路分析** | Hub score, pathway enrichment | 網路醫學 (Exp 21) | 類似 PageRank 的概念，找最重要的節點 |

### 評估指標

| 指標 | 意義 | 我們的結果 |
|------|------|-----------|
| **C-index** | 存活預測的排序準確度（1.0 = 完美，0.5 = 隨機） | 預後簽名 C=0.734 |
| **AUC-ROC** | 二元分類的區分能力 | GNN AUC=0.99, MDM2 依賴 AUC=0.88 |
| **Cohen's d** | 效應量 — 兩組差異有多大（>0.8 算大，>1.2 算非常大） | PTEN-GSS SL: d=−2.00 |
| **FDR** | 多重比較校正後的 p 值（False Discovery Rate） | BCL2 免疫冷腫瘤: FDR=0.028 |
| **Spearman ρ** | 排序相關性 | 計算證據 vs 文獻: ρ=0.50, p=0.0002 |
| **Log-rank p** | 兩組存活曲線是否有顯著差異 | 6 基因簽名: p=6.8×10⁻⁴ |

### 值得注意的技術 insight

1. **GNN AUC=0.99 需要謹慎解讀**：在藥物-基因 link prediction 中達到 AUC=0.99，很可能有 data leakage 或 overfit 的問題。我們對此保持透明。

2. **DeepSurv 在 TCGA 規模下不如 Cox 迴歸**：這是一個重要的反直覺發現 — 當樣本量只有 100-150 時，傳統統計方法比深度學習更穩健。這跟 ML 領域「simple baselines are hard to beat」的現象一致。

3. **LASSO 的穩定性選擇**：我們做了 30 次 bootstrap 的穩定性選擇，結果發現在 TCGA 的樣本規模下，幾乎沒有基因能在 >50% 的 bootstrap 中被選中 → 這是 honest reporting，而不是只報最好的結果。

## 八、這些發現離「治療癌症」還有多遠？

### 藥物開發的完整流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        藥物開發 Pipeline                             │
│                                                                     │
│  ★ 靶點發現    →  靶點驗證  →  先導化合物  →  臨床前  → 臨床試驗 → FDA │
│  (我們在這裡)    (濕實驗)    (藥物化學)    (動物)    (人體)    (核准) │
│                                                                     │
│  ◄── 2026 ──►  ◄─ 2-3年 ─►  ◄─ 2-3年 ─►  ◄ 2-3年 ►  ◄ 5-10年 ► │
│                                                                     │
│  成功率: 100%     ~50%         ~30%        ~20%       ~10%    ~5%   │
└─────────────────────────────────────────────────────────────────────┘
```

### 坦白說

我們的研究處於**最前端的「靶點發現」階段**。我們：

**能做到的**：
- 從 17,386 個基因中篩選出最值得投入資源的靶點清單
- 生成可被實驗檢驗的具體假說（例如：「PTEN 突變的肺癌細胞株 + BSO 處理應產生協同殺傷效果」）
- 為臨床試驗的病人選擇提供計算依據
- 提供已核准藥物的 SL 理論再利用方向

**不能做到的**：
- 不能直接改變臨床治療指引（需要臨床試驗）
- 不能保證計算預測的體內準確性（細胞株 ≠ 人體腫瘤）
- 不能跳過 5-10 年的臨床驗證流程

### 歷史上類似的成功案例

| 研究 | 從計算/發現到 FDA 核准 |
|------|----------------------|
| BRCA → PARP 抑制劑 (Olaparib) | 2005 → 2014（9 年） |
| ARID1A → EZH2 抑制劑 | 2015 → 2025 Phase II（10 年） |
| KRAS → SHP2 抑制劑 | 2018 → 2024 Phase I（6 年） |
| **我們的 PTEN → 鐵死亡** | 2026 → 需要合作進行體外驗證 |

### 我們的真正價值

我們的工作是**藍圖 (blueprint)**，不是治療 (cure)。但好的藍圖是好建築的必要前提。

整個精準醫療的邏輯鏈是：

```
靶點發現 (我們) → 濕實驗驗證 (合作者) → 藥物設計 → 臨床試驗 → 病人受益
```

沒有第一步，後面都不存在。

## 九、需要什麼樣的濕實驗來驗證？

以下是我們的計算發現要變成高影響力論文、甚至走向臨床，所需要的實驗驗證：

### 優先級 1：GPX4 / 鐵死亡驗證（最具新穎性）

**需要的實驗**：
1. **CRISPR 敲除**：在帶有 BRAF / EGFR / PTEN 突變的細胞株中，用 CRISPR 敲掉 GPX4，觀察是否選擇性致死
2. **藥物處理**：用 RSL3（GPX4 小分子抑制劑）或 BSO（GSH 合成抑制劑）處理突變 vs 野生型細胞株
3. **鐵死亡標記檢測**：脂質過氧化 (C11-BODIPY)、鐵離子水平 (Calcein-AM)、細胞存活率 (MTT/CCK-8)

**預期結果**：PTEN 突變的細胞株對 BSO 的 IC50 顯著低於野生型

**需要的設備**：細胞培養室、CRISPR 基因編輯平台、流式細胞儀

### 優先級 2：SL 預後簽名的前瞻性驗證

**需要的驗證**：
1. 回溯性驗證：在 2-3 個獨立的肺腺癌世代（如 METABRIC、ICGC）中驗證 6 基因簽名
2. 前瞻性：與胸腔腫瘤科合作，在新收案病人中前瞻性測試簽名的預測能力

### 優先級 3：BCL2 + 免疫治療組合

**需要的實驗**：
1. 在免疫冷乳癌細胞株中測試 Venetoclax 是否能改變免疫微環境
2. 小鼠模型：Venetoclax + anti-PD-1 組合 vs 單藥

### 優先級 4：SWI/SNF → PRC2 重塑

**需要的實驗**：
1. 在 SWI/SNF 突變的細胞株中測試 Tazemetostat (EZH2 抑制劑) 的敏感性
2. 與 SWI/SNF 野生型細胞株做對比

### 合作需求一覽

| 需要的能力 | 用途 | 優先級 |
|-----------|------|-------|
| CRISPR 基因編輯 | 驗證 SL 關係 | 最高 |
| 細胞存活率檢測 (MTT/CCK-8) | 量化藥物效果 | 最高 |
| 流式細胞儀 | 鐵死亡標記檢測 | 高 |
| 小鼠腫瘤模型 | 體內驗證 | 中 |
| 臨床腫瘤檢體 | 前瞻性簽名驗證 | 中 |
